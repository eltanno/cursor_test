# Design Assets

## Purpose

This directory stores design assets for the project including:

- UI mockups and wireframes
- Data model diagrams
- Architecture diagrams
- Flow charts
- API design documents

## Subdirectories

### ui-mockups/
- User interface designs
- Wireframes
- Screen flows
- Supported formats: PNG, JPG, SVG, Figma exports

### data-models/
- Database schema diagrams
- Entity relationship diagrams (ERDs)
- Data flow diagrams
- Supported formats: PNG, JPG, SVG, PlantUML, Mermaid

## Naming Conventions

Use descriptive names:
- `login-page-mockup.png`
- `user-database-schema.svg`
- `authentication-flow.png`
- `api-architecture-diagram.png`

## Tools Recommended

- **UI Design**: Figma, Sketch, Adobe XD
- **Diagrams**: Draw.io, Lucidchart, PlantUML, Mermaid
- **Screenshots**: Native OS tools
- **Version Control**: Keep source files if possible (.fig, .sketch, etc.)

## Linking to Designs

Reference designs in planning documents:

```markdown
## Design References

- UI Mockup: [Login Page](../designs/ui-mockups/login-page-mockup.png)
- Data Model: [User Schema](../designs/data-models/user-database-schema.svg)
```
